# ZZU 电量监控 - 前端页面

此分支 (`page`) 包含 GitHub Pages 部署的静态资源，与 `main` 分支分离管理。

## 分支优势

| 特性 | 说明 |
|------|------|
| **独立部署** | 前端资源与后端代码分离，部署更灵活 |
| **数据持久化** | 电量数据存储在此分支，不受代码更新影响 |
| **自动更新** | GitHub Actions 自动推送数据到此分支 |
| **快速加载** | 仅包含静态资源，GitHub Pages 加载更快 |
| **版本隔离** | 前端更新不影响数据，数据更新不影响代码 |

## 文件说明

| 文件 | 说明 |
|------|------|
| `index.html` | 主页面，数据可视化 |
| `style.css` | 样式文件，支持深色/浅色主题 |
| `main.js` | 主要 JavaScript 逻辑 |
| `room.js` | 房间查询器数据和功能 |
| `favicon.ico` | 网站图标 |
| `data/` | 电量数据（JSON 格式） |
| `data/tokens.enc` | 加密的认证令牌（AES-256-GCM） |

## 功能特性

- **ECharts 6.0** 图表展示电量趋势
- **深色/浅色主题** 自动跟随系统或手动切换
- **响应式布局** 适配桌面端和移动端
- **玻璃态 UI** 现代化视觉设计
- **房间查询器** 快速查找房间编号，支持全校区
- **年度总结** 支持多年份选择，可导出图片
- **实时统计** 日均消耗、预计可用天数

## 数据格式

```json
// data/2025-01.json
[
  {"time": "01-04 12:00", "light_Balance": 50.5, "ac_Balance": 30.2},
  ...
]
```

## 数据更新

数据由 GitHub Actions 自动更新，可通过 Pipedream 实现精确定时触发。
